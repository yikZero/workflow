Use namespaced streams when different consumers need different views of workflow progress — a UI showing content drafts while an ops dashboard tracks latency.

## Pattern

The workflow delegates all work to step functions, then publishes results to namespace-specific helper steps. Each namespace is independent, so clients subscribe only to the events they care about.

```typescript lineNumbers
declare function buildOutline(topic: string): Promise<string>; // @setup
declare function writeSections(topic: string, outline: string): Promise<string[]>; // @setup
declare function publishDraftChunk(runId: string, text: string): Promise<void>; // @setup
declare function recordTelemetry(runId: string, event: string, value?: number): Promise<void>; // @setup

export async function namespacedStreams(runId: string, topic: string) {
  "use workflow";

  await recordTelemetry(runId, "start");

  const outline = await buildOutline(topic);
  await publishDraftChunk(runId, outline);

  const sections = await writeSections(topic, outline);
  await Promise.all(
    sections.map((section) => publishDraftChunk(runId, section)),
  );

  await recordTelemetry(runId, "done", sections.length + 1);
  return { runId, sectionCount: sections.length + 1 };
}
```

## Key APIs

- [`"use workflow"`](/docs/api-reference/workflow/use-workflow) — declares the orchestrator function
- [`Promise.all()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/all) — publishes independent outputs in parallel
