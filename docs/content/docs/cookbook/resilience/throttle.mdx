Use throttle when you need to limit how many requests go through so you don't exceed an external rate limit.

## Pattern

Maintain a token bucket in the orchestrator. Before processing each request, check for available tokens. Since workflow state is durably persisted, the throttle survives restarts.

```typescript lineNumbers
declare function processRequest(requestId: string): Promise<void>; // @setup

export async function throttleFlow(
  requests: { id: string }[],
  capacity: number,
  refillRate: number
) {
  "use workflow";

  let tokens = capacity;
  let accepted = 0;
  let rejected = 0;

  for (let i = 0; i < requests.length; i++) {
    if (tokens > 0) {
      tokens--;
      accepted++;
      await processRequest(requests[i].id);
    } else {
      rejected++;
    }

    if ((i + 1) % refillRate === 0 && tokens < capacity) {
      tokens++;
    }
  }

  return { accepted, rejected, total: requests.length };
}
```

## Key APIs

- [`"use workflow"`](/docs/api-reference/workflow/use-workflow) — declares the orchestrator function
