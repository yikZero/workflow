Core workflow primitives including steps, context management, streaming, webhooks, and error handling.

## Installation

```package-install
npm i workflow
```

## Workflow-only APIs

These functions must be called inside `"use workflow"` functions:

<Cards>
    <Card href="/docs/api-reference/workflow/sleep" title="sleep()">
        Suspend a workflow for a specified duration or until a date. Deterministic and replay-safe.
    </Card>
    <Card href="/docs/api-reference/workflow/fetch" title="fetch()">
        Make HTTP requests from within a workflow. Runs as a step under the hood.
    </Card>
    <Card href="/docs/api-reference/workflow/create-hook" title="createHook()">
        Create a hook with an optional custom token, resumed server-side via `resumeHook()`.
    </Card>
    <Card href="/docs/api-reference/workflow/create-webhook" title="createWebhook()">
        Create a webhook with a randomly generated token and public HTTP endpoint.
    </Card>
</Cards>

## Workflow and Step APIs

These functions can be called from either `"use workflow"` or `"use step"` functions:

<Cards>
    <Card href="/docs/api-reference/workflow/get-workflow-metadata" title="getWorkflowMetadata()">
        Return context about the current workflow execution.
    </Card>
    <Card href="/docs/api-reference/workflow/get-writable" title="getWritable()">
        Access the current workflow run's stream.
    </Card>
</Cards>

## Error Classes

Error classes that can be thrown inside a step to control retry behavior.

<Cards>
    <Card href="/docs/api-reference/workflow/fatal-error" title="FatalError()">
        When thrown, marks a step as failed and the step is not retried.
    </Card>
    <Card href="/docs/api-reference/workflow/retryable-error" title="RetryableError()">
        When thrown, marks a step as retryable with an optional parameter.
    </Card>
</Cards>
