A workflow that sends an email and waits 3 days for a response. In a traditional system, a worker sits idle for 72 hours. In Workflow DevKit, nothing exists during those 72 hours.

## The Key Insight

Wall-clock time is free because compute exists only at decision points.

A workflow has **decision points** — dispatching steps, collecting results, advancing — and **gaps** between them. Traditional orchestrators keep a process alive during gaps. Workflow DevKit doesn't.

When the workflow hits a step or `sleep()`, it records the pending operation as an event, returns `timeoutSeconds` to the queue, and exits. The compute is freed. No process sleeps, no connection is held.

## The Three Mechanics

Three runtime properties compose into the cost model:

1. **Queue-driven invocation** — workflow handlers are invoked by queue messages, not by long-lived processes. No message, no compute.
2. **Zero-cost suspension** — when a workflow encounters a step, hook, or sleep, it records the pending operations as events, returns, and exits. No thread or connection is held.
3. **Delayed re-enqueue** — for timed waits, the queue delivers the next message after the specified delay. The workflow process does not exist during the gap.

```ts
// The handler returns timeoutSeconds — then compute goes to zero
const minTimeoutSeconds = waitItems.reduce<number | null>(
  (min, queueItem) => {
    const resumeAtMs = queueItem.resumeAt.getTime();
    const delayMs = Math.max(1000, resumeAtMs - now);
    const timeoutSeconds = Math.ceil(delayMs / 1000);
    if (min === null) return timeoutSeconds;
    return Math.min(min, timeoutSeconds);
  },
  null
);
```

When a step completes, the step handler re-enqueues the workflow. A fresh invocation loads the event log, replays through cached results in milliseconds, and resumes at the next decision point. For timed waits, the queue delivers the next message after the delay — no polling, no sleeping process.

## Why It Matters

A `sleep('7d')` costs the same as `sleep('5s')` — both produce a delayed queue message and release all resources. Replay through cached step results takes milliseconds. Your cost is proportional to work performed, not wall-clock time. No always-on workers. No paying to wait.

A week-long workflow. Milliseconds of compute. That's the model.
